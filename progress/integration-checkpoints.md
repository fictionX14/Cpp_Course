# Integration Checkpoints

Cumulative tests/coding problems run at the **end of each phase** — pulling together everything learned in that phase, not just the most recent concept. This is the gate for advancing to the next phase (separate from the day-to-day spaced-repetition review in `review-queue.md`, which handles individual-concept retention).

**Pass condition:** solve/answer using concepts from across the whole phase, not just the last session's material. Fumbles get logged and re-drilled before re-attempting — no advancing on a partial pass.
